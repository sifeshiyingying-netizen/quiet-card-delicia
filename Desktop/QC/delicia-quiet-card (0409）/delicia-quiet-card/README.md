# Delicia Quiet Card — 长文期刊式卡片生成器

**版本**: v1.1.1 | **更新时间**: 2026-04-04

## 产品定位

专为小红书长文系列内容创作者设计的期刊卡片生成工具。实现 **所见即所得 + 最少操作 + 审美稳定** 三大核心价值。

## 卡片类型与结构

```
封面卡片 (cover_card)          → 3种模板 (Magazine/Poster/Minimal) × 5种配色
  ├── Logo + 期刊名称
  ├── 封面图片（可选，居中裁剪压缩）
  ├── 主标题（H1观点）
  └── 期号 + 日期

导读卡片 (guide_card)
  ├── Logo + 期刊名称
  ├── CONTENTS · 本期导读
  └── 核心观点序号列表

正文卡片 (body_card) × N 张
  ├── 期刊名称（顶部）
  ├── H1 / H2 / P 内容块（1.8行高，40px内边距）
  └── 底部装饰线 + 页码

总结卡片 (summary_card)
  ├── KEY TAKEAWAYS
  └── 核心观点 ✓ 清单

CTA 卡片 (cta_card)
  ├── 纯色背景（主题色）
  ├── 你的思考
  └── 互动引导 + 账号
```

## 设计规范

### 布局规格（严格符合）
- 顶部工具栏高度: 60px
- 左侧边栏宽度: 380px
- 手机模拟框宽度: 390px（刘海 120×28px）

### 颜色系统
- **品牌色**: `#7C8C3E` 橄榄绿
- **画布背景**: `#FDFCFA`
- **侧边栏**: `#F7F5F2`
- **主色文字**: `#37352F`

### 字体规范（均为版权安全字体）
- 中文正文: `Noto Serif SC`（经典/文艺）/ `Noto Sans SC`（现代）
- 英文展示: `Playfair Display`（杂志）/ `Inter`（现代）/ `Dancing Script`（手写）
- **严禁**: 方正、汉仪、微软雅黑、苹方、Didot、Garamond

### 排版节奏
- 内边距: 40px
- 行高: 1.8（正文）/ 1.4（标题）
- 字间距: 0.025em（正文）/ 0.08em（封面）

## 技术栈

- **框架**: React 18 + TypeScript
- **构建**: Vite 7
- **样式**: Tailwind CSS 4 + CSS Variables
- **动画**: Framer Motion
- **导出**: html2canvas + JSZip
- **状态**: React Context + useReducer
- **历史管理**: 自定义 HistoryManager 类（50步双向撤销）

## v1.2.0 新功能

### F010 排版风格三模式正式实装
三种排版风格现在实际影响正文卡片布局（之前仅追踪状态，未影响渲染）：
- **Quiet**（静谧纸质感）：期刊名左 + 页码右（淡化）、底部强调色条 + 灰线
- **Editorial**（杂志风格）：左侧5px垂直色条、顶部强调色背景标题栏、加粗期刊名/页码、编辑感排版
- **Soft**（暖色调）：居中期刊名 + 两侧短横线，中间圆点 + 页码，底部旋转菱形装饰

### F012 自定义配色（Custom Color）
主题配色选择器末尾新增"自定义"按钮（虚线边框）：
- 点击后展开色号输入面板
- 纸张底色 + 强调色各有原生取色器（色块点击）+ 十六进制手动输入框（#RRGGBB 格式）
- 输入无效格式时输入框变红提示
- 修改即时同步到卡片预览

## v1.1.1 Bug 修复

### F047 maxCharCalculator.ts — 分页字符数计算修正
- **问题**: 旧代码基于 1080px 渲染宽度估算，实际渲染宽度为 360px，导致每页最大字符数约为 935，远超实际可容纳量（约 196）
- **影响**: 正文被全部挤入一页，卡片底部文字被 `overflow:hidden` 截断
- **修正**: 重新基于 RENDER_WIDTH=360px + BodyCard 实际开销 109px 精确计算
  - 4:3 → 480px 高 → 196 chars/页（原 ~935，误差 4.8×）
  - 15:9 → 600px 高 → 272 chars/页
  - 1:1 → 360px 高 → 136 chars/页
- **验证**: 新增 8 项单元测试（vitest），全部通过
